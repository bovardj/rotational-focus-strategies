## Phase 1: Performance Fixes
 
**[ ] 1. Fix RLS Initialization Plan warnings**
 
Use Claude Code with the Supabase MCP connected to find and fix all RLS policies that re-evaluate `auth.jwt()` per row. The fix is wrapping the call in a `SELECT`:
 
```bash
claude "Using the Supabase MCP, find all RLS policies on the following tables that call auth.jwt() or auth.<function>() without wrapping them in a SELECT: assigned_strategies, baseline_survey_responses, daily_survey_responses, days_completed, days_expected, end_survey_responses, scheduled_notifications, subscriptions, user_strategies, users. For each affected policy, propose updated SQL that wraps the auth call in (SELECT auth.<function>()) for better performance. Do not apply any changes yet — proposals only."
```
 
Review and approve the proposed SQL, then apply:
 
```bash
claude "Apply the following approved RLS policy changes and nothing else: [paste approved SQL here]"
```

## Phase 3: Supabase API Key Migration
 
> **Note:** The legacy `anon` and `service_role` keys work until end of 2026 — this is not urgent but should be done before then.
 
**[ ] 6. Migrate environment variable references in code**
 
```bash
claude "Migrate all Supabase environment variable references from the legacy key format (NEXT_PUBLIC_SUPABASE_ANON_KEY, SUPABASE_SERVICE_ROLE_KEY) to the new Supabase key format (NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY, SUPABASE_SECRET_KEY). Find all references, update them, and list every file changed."
```
 
**[ ] 7. Get the new key values from Supabase**
- Go to **Settings → API Keys** in your Supabase project
- Click **Create new API Keys** if you don't have a publishable key yet
- Copy the **Publishable key** and **Secret key**
**[ ] 8. Update Vercel environment variables**
- Go to your Vercel project → **Settings → Environment Variables**
- Replace the old variables with the new ones:
| Old Variable | New Variable |
|---|---|
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` |
| `SUPABASE_SERVICE_ROLE_KEY` | `SUPABASE_SECRET_KEY` |
 
**[ ] 9. Redeploy and test**
- Trigger a manual redeploy on Vercel
- Test auth, database access, and any protected routes
